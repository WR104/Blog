The kth Permutation Sequence

This problem asks us to find the kth permutation sequence of numbers from 1 to n. The approach taken is to divide the problem into smaller sub-problems and solve them one at a time. The basic idea is to start with an array of numbers from 1 to n and eliminate numbers one by one based on the given k value.

Let’s take an example to understand this approach. Say, we need to find the 5th permutation sequence of the numbers from 1 to 3. The permutations for this array would be:

  • 123
  • 132
  • 213
  • 231
  • 312
  • 321

From this list, we need to find the 5th permutation, which is “312”. To solve this problem programmatically, we can use the following steps:

  1. Create a vector with numbers from 1 to n.
  2. Calculate the factorial of n and subtract 1 from k.
  3. For each i from n to 1, calculate the index j such that j = k / (i-1)! and subtract j*(i-1)! from k.
  4. Remove the number at index j from the vector and append it to the result.
  5. Repeat steps 3 and 4 until the vector is empty.

Let’s go through these steps for the example of finding the 5th permutation sequence of the numbers from 1 to 3.

  1. Create the vector [1, 2, 3].
  2. n=3, so the factorial of n is 6. Subtracting 1 from k=5, we get k=4.
  3. For i=3, j=k/(i-1)!=4/(3-1)!=2. Subtracting j*(i-1)!=2*(3-1)!=4 from k=4, we get k=0.
  4. Remove the number at index j=2 from the vector, which is 3, and append it to the result.
  5. The vector is now [1, 2]. For i=2, j=k/(i-1)!=0/(2-1)!=0. Subtracting j*(i-1)!=0*(2-1)!=0 from k=0, we get k=0.
  6. Remove the number at index j=0 from the vector, which is 1, and append it to the result.
  7. The vector is now [2]. For i=1, j=k/(i-1)!=0/(1-1)!=0. Subtracting j*(i-1)!=0*(1-1)!=0 from k=0, we get k=0.
  8. Remove the number at index j=0 from the vector, which is 2, and append it to the result.
  9. The vector is now empty and we have found the 5th permutation sequence, which is “312”.

This algorithm has a time complexity of O(n^2) because we are calculating factorials in each iteration. However, we can optimize it to O(n) by using a pre-calculated factorial array instead of computing it every time.
